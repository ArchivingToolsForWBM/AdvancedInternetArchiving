This tool will take a given tab-seperated values (TSV) table from google sheets of the archive results (“Archive URLs”), and searches for any potential issues or anomalies. Note: this tool only judges based on the output data of the table, not the actual web-page content.

How to use (click to open)
  1. Have a google account and go to google drive, create a spreadsheet anywhere you wish.

  2. Make sure you authorized WBGS the ability to edit that spreadsheet, else it will not be able to write the results. To do that, right-click the spreadsheet file (or the folder it is in) → Share → Under “General Access” set it to “Anyone with the link” and the role (a dropdown list to the right) set to “Editor”. However, even easier, have the folder with that share setting with the google sheet for processing URLs inside of that.

  3. Open the spreadsheet, and have your list of URLs all on column A (the first column) where each row contains 1 URL, with the first URL on the first row.

  4. Before you start a process and any subsequent process, make sure the spreadsheet is saved! This is especially for those with weak internet connection.

  5. On Wayback Gsheets, make sure you are logged in (both the IA and google), and on the main menu, select “Archive URLs”, paste your URL of the google sheets containing the list of URLs, check the “Save results in a new Sheet.” option and press “Archive”.

    In case you keep forgetting to check the option to have the results in a new sheet, I made a rather simple greasemonkey script in AdvancedInternetArchiving/GreasemonkeyScripts/WBGS. It automatically checks the checkbox of that option when being on the start process page, along with other features.
    • The reason why I instructed you to have “Save results in a new Sheet.” checked is because of a bug I encountered that a process may duplicate, and overwrite any save results by the second and subsequent duplicate processes. This bug usually happens some time after the process have started after it initally-queues, or finished. When this happens, you get 2+ identical emails titled “Wayback gsheets capture tracking” sharing the same tracking URLs linked, and the home page of WBGS after logging in also showing duplicate processes. When having the results in a new sheet and the process duplicates, they won't overwrite each other since each process will create its own sheet.

  6. If the process successfully starts, you'll get an email titled “Wayback gsheets capture tracking” about that process.
    • If you have Save results in a new Sheet. checked, I strongly recommend clearing the first sheet (delete the contents on the sheet) after receiving the email and not touch the new sheet's until the process is done. This prevents future process duplication bugs from re-traversing URLs. It will leave behind empty extra sheets with a title being the UTC timestamp each time a duplication occurs. WBGS makes a copy of the URL list internally when the traversing starts so editing/clearing the list on google sheets afterwards will not affect the list on WBGS.

  7. In the newly created sheet (created by the WBGS when the process starts traversing URLs), when URLs get a result (it updates every N URLs processed as it processes), the spreadsheet should be in this format: 
    SentURL1	SaveBeforeStatus	HTTPStatus	ArchiveURL/Error	Outlinks_Capture_Count	FirstArchive
SentURL2	SaveBeforeStatus	HTTPStatus	ArchiveURL/Error	Outlinks_Capture_Count	FirstArchive
SentURL3	SaveBeforeStatus	HTTPStatus	ArchiveURL/Error	Outlinks_Capture_Count	FirstArchive
...

    After the process finishes (you should get an email saying “Wayback gsheets capture report” if it did), or if a process queues or gets stuck on the remaining few URLs, select columns A-F select the cells covering columns A-F and copy (CTRL+C). Then on this HTML file, paste (CTRL+V) your list in the following textarea. The data should be converted into tab-seperated values.

  8. This HTML file will now display various stuff (at the bottom), mainly to filter to only show URLs that probably haven't saved. By default, it will get only URLs that are “potentially not saved properly”: errors, redirects, or strange things like column D being blank, etc.. From this point on, you can re-attempt to save these URLs again. An easy-to-copy URLs is provided in an output textarea.

    • If you happen to run into issues with the process (queued mid-process, stuck on last few URLs, and/or you aborting the process), I strongly recommend starting a new google sheet and use that instead because there is a tendency that a problematic process may run again (duplicate) after a queue, and if you have edited the sheet while there's a process on it, could overwrite the save results.


  9. If you run into process duplicating itself and both are done, I would rather have you filter only URLs flagged as “success” on all save result sheets, have them all formatted as Display just entered URLs in the plaintext, copy that, open the list cleaner tool (provided here at "JS tools/List cleaner") paste them into the blacklist, and copy all the URLs you attempted to save and paste them into the first textarea (make sure no tabs and one URL per line on both textareas). This will remove any URLs that have been successfully saved at least once on any processes including duplicates. Then re-attempt saving the URLs output from the list cleaner.

I strongly recommend checking the process and emails to see any anomalies, this include anything strange happening on the process activity log, using the browser inspect element for any http error statuses on the WBGS pages, etc. If you do find one, make sure you contact the IA about it.
